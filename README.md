# blog-database-github-client

A lightweight, zero-dependency SDK for reading content stored as JSON and Markdown files on GitHub. Works in browsers and Node.js (≥18).

Generated by the [Static CMS](https://github.com/sumitpal29/static-cms) local editor and published to your own GitHub repo.

## How it works

Your content lives in a GitHub repository as flat files generated by the Static CMS backend:

```
<repo>/
└── <project>/
    ├── meta/
    │   ├── index.json          ← pagination index
    │   ├── list_1.json         ← page 1 post metadata
    │   └── list_2.json         ← page 2 post metadata
    ├── posts/
    │   └── my-first-post.md   ← full post with front-matter
    └── books/
        ├── index.json          ← list of all books
        └── my-book/
            ├── map.json        ← chapter/folder tree
            └── chapter-one.md ← book file with front-matter
```

The SDK fetches these files directly from `raw.githubusercontent.com` — no server, no API key.

## Installation

```bash
npm install blog-database-github-client
```

## Quick start

```js
import { createBlogClient } from 'blog-database-github-client';

const client = createBlogClient({
  repo: 'your-username/your-repo',
  branch: 'main',
  project: 'my_blog',
});

// Fetch page 1 of posts
const { items, totalPosts } = await client.getPosts(1);

// Fetch a single post by slug
const { frontmatter, content } = await client.getPost('hello-world');

// Search across all posts
const results = await client.search('javascript');

// List all books
const books = await client.getBooks();

// Fetch a book's chapter tree
const map = await client.getBookMap('my-book');

// Fetch a specific chapter
const chapter = await client.getBookFile('my-book', 'chapter-one.md');
```

## API Reference

### `createBlogClient(config)`

| Option | Type | Required | Description |
| --- | --- | --- | --- |
| `repo` | `string` | ✓ | GitHub repo in `owner/repo` format |
| `branch` | `string` | ✓ | Branch name (e.g. `"main"`) |
| `project` | `string` | ✓ | Project folder inside the repo |
| `cacheTtl` | `number` | | Cache TTL in ms (default: 5 min) |
| `useLocalStorage` | `boolean` | | Persist cache to `localStorage` in browsers (default: `true`) |

Returns a `BlogClient` instance.

---

### Posts

#### `client.getPosts(page?)`

Fetches one page of post metadata. Use this for paginated listing UIs.

```js
const { page, totalPosts, items } = await client.getPosts(1);
// items: Array<{ slug, title, description, createdAt, updatedAt, metatags }>
```

#### Pagination — fetching all pages

Use `getIndex()` to know the total number of pages, then loop:

```js
// Ensure the index is loaded first
await client.getPosts(1);
const index = client.getIndex();
// index: { totalPosts, totalPages, pages, generatedAt }

for (let p = 1; p <= index.totalPages; p++) {
  const { items } = await client.getPosts(p);
  // render items for page p
}
```

Or fetch everything at once in parallel:

```js
const allPosts = await client.getAllPosts();
```

---

#### `client.getAllPosts()`

Fetches all posts across all pages in parallel. Good for search indexes or sitemaps.

```js
const posts = await client.getAllPosts();
```

---

#### `client.getPost(slug)`

Fetches the full content of a post by slug.

```js
const { slug, frontmatter, content } = await client.getPost('my-post-slug');
// content: raw Markdown string
```

---

#### `client.search(query)`

Client-side full-text search. Searches `title`, `description`, and `metatags`. All terms must match (AND logic).

```js
const results = await client.search('typescript tips');
```

---

#### `client.getIndex()`

Returns the cached pagination index without a network request. Returns `null` if not yet loaded (call `getPosts` first).

```js
const index = client.getIndex();
// { totalPosts, totalPages, pages, generatedAt }
```

---

### Books

#### `client.getBooks()`

Lists all books in the project. Reads from `books/index.json` generated by the CMS.

```js
const books = await client.getBooks();
// [{ slug, name, description }, ...]
```

---

#### `client.getBookMap(bookSlug)`

Fetches the full file/folder tree for a book. Reads from `books/<bookSlug>/map.json`.

```js
const map = await client.getBookMap('my-book');
// map.tree.items — array of { type: 'file'|'folder', name, path, items? }
```

---

#### `client.getBookFile(bookSlug, filePath)`

Fetches a single book file's content and front-matter.

```js
const { content, frontmatter } = await client.getBookFile('my-book', 'concepts/overview.md');
```

---

### Cache

#### `client.clearCache()`

Flushes all in-memory (and `localStorage`) cache for this client instance.

---

## Caching

Responses are cached in memory with a default TTL of 5 minutes. In browser environments, entries are also persisted to `localStorage` so they survive page reloads within the TTL window. Pass `useLocalStorage: false` to disable this.

## Requirements

- **Runtime**: Node.js ≥ 18, or any modern browser (uses native `fetch`)
- **ESM only**: Use `import`, not `require`
